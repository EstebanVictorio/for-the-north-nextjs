How to React really fast - Part 1: Pages

Have you ever found yourself in practicing some React, in your environment, but felt like it was way too much setup for some simple practicing? No worries.

NextJS got you covered.

How come do we need to Webpack everytime you ask? That might be the case, but that particular task doesn't everytime land on you, dear developer. We just need to reach out our research a little longer or further. Enter NextJS.

NextJS is a React framework that allows you to deliver your React apps faster.

Let's jump into it. You never buy someone's idea or suggestion without a showcase of it. So... Let's say we want to build just a veeery basic but descriptive page about a product. This might be composed of the following:

  • A Home page, describing highlights of the product.
  • An About page, describing why this product exists.
  • A Contact page, letting you know the whereabouts of theproduct's owners and how you might acquire the product.

Before getting onto this, there are two pre-requisites that you need to have installed on your machine:

  • Node JS
  • npm/yarn

Being able to start is pretty easy. Open the editor of your choice, choose a folder to work from, and open a terminal on your computer as well. After these minor steps, run any of the following:

npm i --save next react react-dom

Or, if you prefer...

yarn add next react react-dom

Now, before beginning to code our pages, there is a tiny step we should do if we do not want to use the terminal (very often). NextJS allows you to start a server which serves your app and your pages along with it, as well as helping you with hot reloading so you don't have to start the server as many times as you would do everytime you performn an update to your code. To do this, we can add the following line to the scripts field in package.json:


  {
    "dependencies":{
      "react": "16.8.6",
      "react-dom": "16.8.6",
      "next": "8.1.0"
    },
    "scripts": {
      "dev" : "next"
    }
  }
This config allows us to run NextJS in development mode, so we can run yarn dev in the terminal. This is a preference and you might as well use yarn next.

Now we're all set to start. The following step is to create a folder named pages, as well as a layout component in our root directory, so we create the following file structure:


    node_modules
    yarn.lock
    package.json
    layout.js
    /pages
      index.js
      about.js
      contact.js

Now, we can proceed to code our very first NextJS-based React app. As our main component driving navigation towards the app, we'll build the Layout component:


    import Link from 'next/link'
    const Layout = ({ children }) => (
      <main>
        <nav>
          <ul>
            <li>
              <Link href="/">
                <a>Home</a>
              </Link>
            </li>
            <li>
              <Link href="/about">
                <a>About</a>
              </Link>
            </li>
            <li>
              <Link href="/contact">
                <a>Contact</a>
              </Link>
            </li>
          </ul>
        </nav>
        { children }
      </main>
    )
  
    export default Layout

Let's dive a little into what happens here. First, we need a way to navigate across the site. Of course, we could do this purely with anchor tags, but since we are in a framework, we'll take advantage of the Link component that NextJS provides. This component allows the navigation as if it were an single page app. So, loading will look seamless. Currently, NextJS encourages you to still enclose the text content inside an anchor tag due to deprecation using text-only, but that makes no harm to the navigation.

Something to point out here is that we're using a very particular syntax here, called destructuring assignment. We're not diving here too deep into that, but this is a feature of the ES6 standard, which lets you unwrap the property values you send/inject to a component in an attribute-like manner in your code. Since React wraps everything into the props argument, using destructuring the props object will give you access to all of the properties you provided. If you try to unwrap a property that's not injected, you will get the undefined value. This is JavaScript behavior itself, not a React one.

One last thing here that allows us to have the navelement in all pages is the property children. This property represents the elements that are wrapped inside a component when used:


  // Layout component in layout.js
  import Layout from './../layout'

  const Home = () => (
    <Layout> // Wraps 'div'.
      <div>Home page</div> // <-- Children set
    </Layout>
  )

  export default Home

  // Home page component in /pages/index.js
  import Link from 'next/link'
  const Layout = ({ children }) => ( // <--- Unwraps children via destructuring
    <main>
      <nav>
        <ul>
          <li>
            <Link href="/">
              <a>Home</a>
            </Link>
          </li>
          <li>
            <Link href="/about">
              <a>About</a>
            </Link>
          </li>
          <li>
            <Link href="/contact">
              <a>Contact</a>
            </Link>
          </li>
        </ul>
      </nav>
      { children } // <--- Children rendered
    </main>
  )

  export default Layout

Now, for our first page, there's index.js, which will respond automatically to the / route. We would write something like:


    import Layout from './../layout'

    const Home = () => (
      <Layout>
        <h1>Distraction is over. Meet: The super lettuce.</h1>
        <p>
          A bio-engineered super lettuce capable of super concentration powers.
        </p>
      </Layout>
    )
  
    export default Home
This would tell NextJS that this is our first page module.

Then, we'll write our /about page code, describing the product:


    import Layout from './../layout'

    const About = () => (
      <Layout>
        <h4>
          About the super lettuce.
        </h4>
        <div>
          The super lettuce will grant you inimaginable concentration powers.
          You will be able to program for hours and not losing a single drop of
          focus in your entire rush. It is proposed to combat one of many problems
          in the world: Concentration Loss.
        </div>
      </Layout>
    )

    export default About

And finally, the last page, where we'll let our users to check where they can reach the company out, the /contact page:


    import Layout from './../layout'

    const Contact = () => (
      <Layout>
        <div>Contact us!</div>
        <a href="mailto:sample@superlettuce.com">
          Reach us at our company email!
        </a>
      </Layout>
    )

    export default Contact

After this, we can run the command we set at thescripts field in package.json, which is npm dev or yarn dev, depending on your package manager. Running that command will lead you to having the following structure:


    .next // This is new!
    node_modules
    yarn.lock
    package.json
    layout.js
    /pages
      index.js
      about.js
      contact.js

When that folder appears, it means that Next built your app, which means that you also got some messages like the following:


    [ wait ]  compiling ...
    [ ready ] compiled successfully
    [ info ]  ready on http://localhost:3000

This means that your app is ready to be used, and you can access it via entering http://localhost:300 in the address bar, which will lead to something like this:

What happened is that Next took care of using Webpack under the hood for the required dependencies to create and develop our app, so we can completely focus on the features rather than the setup. This is a way we can at least get some momentum on diving deep in case we want to learn further or analyze our options at the time of choosing how we want to React in different use cases. This one of those, and it lets us step forward relatively fast.

And that's it! You created a web app that has a tiny setup letting you develop your features in a comfortable way!
